Cette page résume les évolutions marquantes de Claude Code et de l'écosystème Claude au premier semestre 2026. L'orientation générale est claire : Claude Code passe d'un outil que l'on **ouvre, supervise et referme** à un agent capable de **rester actif plus longtemps**, de **toucher davantage du cycle de vie logiciel**, et de **présenter son travail** sous une forme inspectable (graphiques, diagrammes, apps).

> Les notes de version évoluent quasi quotidiennement. Pour la version à jour, voir [Sources & ressources externes](/wiki/actualites/sources-ressources).

## Le modèle par défaut : Claude Opus 4.8

**Claude Opus 4.8** est devenu le modèle le plus avancé disponible publiquement et le **modèle par défaut** de Claude Code (déployé notamment pour les comptes Max, Team Premium, Enterprise pay-as-you-go et API).

À retenir côté modèles : Opus 4.8 est accompagné d'un **« fast mode »** plus rapide et à tarif plus compétitif, pensé pour les itérations rapides où la latence compte. La famille de modèles récents inclut aussi Opus 4.7, Opus 4.6, Sonnet 4.6 et Haiku 4.5.

## Les grandes capacités agentiques

### Computer use depuis le CLI
Claude peut lancer des applications natives et naviguer dans des interfaces graphiques directement depuis le terminal. Cela élargit nettement ce que l'agent peut « toucher » au-delà du seul code.

### Auto mode
Un mode qui automatise les demandes de permission pour les actions jugées sûres tout en bloquant les actions risquées. Objectif : réduire la supervision constante sans sacrifier la sécurité. (Disponible en aperçu, y compris dans l'extension VS Code.)

### Remote control / continuité à distance
La possibilité de garder la main sur des sessions à distance, dans la logique d'agents qui tournent en continu et que l'on rejoint seulement quand une décision humaine est nécessaire.

### Scheduled tasks (tâches planifiées)
Claude Code peut exécuter des travaux logiciels **récurrents** de façon planifiée. Couplé à l'auto mode et au remote control, cela transforme l'outil pour la maintenance régulière (revue, mises à jour, vérifications).

### Cloud auto-fix
Claude prolonge son rôle **après l'ouverture d'une pull request** : il peut corriger automatiquement certains problèmes côté cloud, étendant l'agent au-delà de la simple génération de code.

## Orchestration multi-agents & workflows

### Dynamic workflows
On peut **orchestrer plusieurs sous-agents depuis un seul script**. Un agent « chef » délègue à des sous-agents spécialisés, chacun avec son propre modèle, son prompt et ses outils.

### Multiagent orchestration (Claude Platform)
Sur la plateforme Claude, un agent principal peut déléguer à des sous-agents spécialisés travaillant **en parallèle sur un système de fichiers partagé**. L'ensemble du flux est traçable dans la console Claude.

### Agent view
Une **vue unique pour gérer toutes tes sessions Claude Code** : démarrer des agents, les envoyer en arrière-plan, consulter leur statut et leur dernière réponse, et ne revenir dans une session que lorsqu'une saisie est requise.

### Le skill `/batch`
Exemple emblématique du système de skills : il orchestre des changements à grande échelle dans une base de code **en parallèle**. Il analyse le code, découpe le travail en 5 à 30 unités indépendantes, présente un plan pour approbation, puis lance des agents en arrière-plan dans des **worktrees Git isolés**. Chaque agent implémente son unité, lance les tests et ouvre une pull request.

## Mémoire, skills, hooks

### « Dreaming » (mémoire qui s'améliore entre les exécutions)
Un processus planifié qui **revoit les sessions passées**, repère des motifs et **organise la mémoire** : erreurs récurrentes, workflows partagés et préférences d'équipe sont intégrés à un magasin de mémoire plus utile. Les agents s'améliorent ainsi d'une exécution à l'autre.

### Skills
Le framework de skills permet d'étendre Claude Code avec des commandes et capacités personnalisées. La bibliothèque de skills fournie a été enrichie, et la **découverte automatique** depuis des dossiers imbriqués améliorée. Les skills supportent la substitution de variables (`$ARGUMENTS`, `$ARGUMENTS[N]`, `${CLAUDE_SESSION_ID}`) pour des comportements dynamiques par session.

### Hooks & contexte
Améliorations sur les hooks, la mémoire et le contexte, ainsi que sur les limites de sortie et l'infrastructure de plugins, pour fiabiliser les workflows longs.

### Plugin de sécurité
Un plugin/guidage de sécurité qui **revoit le code à la recherche de vulnérabilités** de façon proactive.

## Présentation du travail : visualisations

Claude peut désormais produire des **graphiques interactifs, des diagrammes, des visualisations et même des interfaces mobiles interactives**. La logique de fond : les agents ont besoin d'interfaces **inspectables** plutôt que de blocs de texte. Terminer la tâche n'est qu'une partie du travail — il faut aussi **packager le résultat** dans une forme que l'on peut inspecter, partager et commenter rapidement.

## Limites & échelle

- **Limites de débit doublées** sur Claude Code, et limites API relevées pour Opus, pour permettre de construire de façon plus fiable à grande échelle.
- Meilleure **visibilité des limites de débit** et statut d'usage plus clair (ex. `claude agents --json` indique ce qui bloque une session en attente).
- De nombreux correctifs sur la connexion, la reprise de session (`resume`), les worktrees, Windows et la stabilité générale.

## Installation : ce qui a changé

- **Installeur natif (recommandé)** — sans dépendance, **pas besoin de Node.js**, mise à jour automatique en arrière-plan :
  - macOS / Linux / WSL : `curl -fsSL https://claude.ai/install.sh | bash`
  - Windows (PowerShell) : `irm https://claude.ai/install.ps1 | iex`
  - Homebrew (macOS/Linux) : `brew install --cask claude-code` (pense à `brew upgrade` pour mettre à jour)
- **npm (toujours supporté, méthode historique)** — pour épingler une version précise. Nécessite **Node.js 18+** :
  - `npm install -g @anthropic-ai/claude-code`
  - Mettre à jour avec `npm install -g @anthropic-ai/claude-code@latest` (évite `npm update -g`).
- **Compte requis** : Claude Pro, Max, Team, Enterprise ou un compte Console (API). Le plan **gratuit de Claude.ai n'inclut pas Claude Code**.
- **Diagnostic** : `claude doctor` détecte automatiquement la plupart des problèmes de configuration.
