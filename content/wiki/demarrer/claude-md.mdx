Le `CLAUDE.md` contient les **instructions et le contexte** que Claude charge au démarrage de chaque session. C'est le levier de [prompt engineering](/wiki/prompt-engineering) le plus rentable dans Claude Code : plus il est clair, moins tu répètes de contexte, et mieux l'agent choisit ses [outils](/wiki/outils). (À distinguer de [`settings.json`](/wiki/cli/settings-json), qui est de la configuration, pas des instructions.)

## Comment Claude découvre `CLAUDE.md`

Claude Code cherche le fichier à plusieurs niveaux, et **concatène** tout ce qu'il trouve dans le contexte système (le plus proche du projet pèse le plus) :

1. `./CLAUDE.md` — racine du projet courant
2. `./CLAUDE.local.md` — overrides personnels (à `.gitignore`)
3. Les `CLAUDE.md` des dossiers parents jusqu'à `$HOME`
4. `~/.claude/CLAUDE.md` — préférences globales utilisateur

> Utilise [`/memory`](/wiki/slash-commands) dans le REPL pour voir exactement ce qui est chargé, et [`/init`](/wiki/slash-commands) pour générer un premier `CLAUDE.md` en analysant le projet.

## Que mettre dans un CLAUDE.md

Garde-le **court (300–800 mots)** et structuré :

- **Stack & architecture** : langages, frameworks, structure des dossiers.
- **Commandes** : dev, build, test, lint.
- **Conventions** : nommage, style, règles (« pas de `any` en TS »).
- **Pièges** : ce qui casse souvent, ce qu'il ne faut pas toucher.

```markdown
# Mon SaaS

Application SaaS de gestion de tâches multi-tenants.

## Stack
- Next.js 16 App Router + TypeScript strict
- Supabase (Auth + Postgres + RLS)

## Commandes
- `pnpm dev` · `pnpm test` · `pnpm build`

## Conventions
- Composants PascalCase dans `components/`
- Server Components par défaut ; `'use client'` seulement si nécessaire
- Toujours typer ; éviter `any`

## À éviter
- Ne pas modifier `supabase/migrations/` à la main → `pnpm db:migrate`
- Pas de `console.log` livré (utiliser `lib/logger.ts`)
```

Tu peux référencer d'autres fichiers via `@chemin/fichier` (`Voir @docs/STYLE_GUIDE.md`) — Claude les lit au besoin sans surcharger le contexte initial.

## Les sections qui rapportent le plus

| Section | Pourquoi |
|---|---|
| **Commandes** | Claude utilise la bonne commande du premier coup |
| **À éviter** | Évite les régressions sur du code legacy |
| **Conventions** | Style cohérent dès la première écriture |
| **Patterns importants** | Architecture respectée sans rappel constant |

## Erreurs courantes

- **Trop long** : un `CLAUDE.md` de 5000 mots pollue le contexte. Découpe en fichiers thématiques référencés via `@`.
- **Trop vague** : « respecter les bonnes pratiques » n'aide pas. Préfère « pas de `any`, utiliser `unknown` si le type est inconnu ».
- **Pas à jour** : un fichier obsolète induit Claude en erreur de façon active.
- **Mélange instructions / doc** : `CLAUDE.md` = instructions à Claude, pas explication du projet aux humains (c'est le rôle du README).

> Anthropic ne publie pas le prompt système interne de Claude Code. On personnalise via `CLAUDE.md` ou le flag `--append-system-prompt`, pas en essayant de « deviner » le prompt système.

## La compaction du contexte

Une session longue accumule tout : lectures de fichiers, sorties d'outils, raisonnements intermédiaires, impasses. Passé environ deux tiers de la fenêtre de contexte, le rapport signal/bruit se dégrade et la qualité baisse — **pas** parce que le modèle a changé, mais parce que le contexte est saturé.

Claude Code **compacte automatiquement** la conversation autour de 95 % de capacité (résumé du passé pour libérer de la place). Tu peux :

- compacter manuellement avec [`/compact`](/wiki/slash-commands) ;
- désactiver la compaction auto avec `DISABLE_AUTOCOMPACT=1` ([variables d'environnement](/wiki/cli/variables-env)) ;
- **déléguer** les recherches lourdes à des [subagents](/wiki/subagents/creer-subagent-custom) (qui lisent dans leur propre contexte et ne remontent qu'un résumé).

## Le vrai principe : contexte propre > gros contexte

Beaucoup pensent que « plus de données dans le contexte = meilleures réponses ». C'est faux : un contexte **ciblé et structuré** bat un gros contexte bruité. C'est aussi pourquoi, pour une base de connaissances ou un chatbot, l'**indexation** (récupérer les bons morceaux) compte plus que le volume brut. Le `CLAUDE.md` et les [subagents](/wiki/subagents) sont tes deux outils pour garder ce contexte propre.
