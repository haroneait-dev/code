La famille Claude 4.x propose trois tiers principaux : **Opus**, **Sonnet** et **Haiku**. Chacun cible un compromis différent entre qualité, vitesse et coût. Cet article synthétise les différences pour t'aider à arbitrer rapidement, sans relire toute la documentation.

## Vue d'ensemble

| Critère | Opus 4.7 | Sonnet 4.6 | Haiku 4.5 |
|---|---|---|---|
| ID modèle | `claude-opus-4-7` | `claude-sonnet-4-6` | `claude-haiku-4-5-20251001` |
| Niveau | Premium | Standard | Compact |
| Qualité raisonnement | Excellent | Très bon | Bon |
| Vitesse | Lente | Moyenne | Rapide |
| Coût input/output | Élevé | Moyen | Faible |
| Contexte | 200K | 200K | 200K |
| Extended thinking | Oui (budget large) | Oui | Oui (budget réduit) |
| Vision | Oui | Oui | Oui |
| Tool use | Excellent | Excellent | Bon |

| Cas d'usage | Choix recommandé |

## Latence typique

| Modèle | Premier token | 1000 tokens générés |
|---|---|---|
| Opus 4.7 | ~1-2s | ~20-40s |
| Sonnet 4.6 | ~500ms-1s | ~10-20s |
| Haiku 4.5 | ~300-600ms | ~3-6s |

> Ces ordres de grandeur valent pour des prompts moyens (~5k tokens d'entrée), sans extended thinking.

## Cascade : combiner les trois

Un pattern productif est la **cascade** :

```ts
async function answer(query: string) {
  // 1. Haiku trie
  const intent = await classify(query); // claude-haiku-4-5-20251001

  if (intent === "simple") {
    return await haikuAnswer(query);
  }

  if (intent === "code") {
    return await sonnetAnswer(query); // claude-sonnet-4-6
  }

  // 2. Cas complexe → Opus
  return await opusAnswer(query); // claude-opus-4-7
}
```

Cette approche optimise le coût : la majorité des requêtes s'arrêtent à Haiku.

## Choisir par contrainte dominante

| Contrainte dominante | Choix |
|---|---|
| Latence < 1s | Haiku |
| Coût < $0.001 / requête | Haiku |
| Qualité maximale | Opus |
| Compromis | Sonnet |
| Code quotidien | Sonnet |
| Recherche / agents complexes | Opus |

## Erreurs courantes

- **Prendre Opus par défaut** : surcoût significatif sans gain de qualité dans 80 % des cas.
- **Prendre Haiku pour du code complexe** : taux d'erreur trop élevé.
- **Ignorer le prompt caching** : sur Sonnet/Opus, les gains de coût sont substantiels sur les prompts répétitifs.
